== Executive Summary

Codebase-gradle is evolving from a **generic code-generation plugin** to a **production-grade Assistant Protocol** for autonomous code generation and pedagogy. This document consolidates the strategic vision, technical architecture, roadmap, and stabilization requirements.

**Core Vision:** Build a stable, versionable assistant protocol (v1.0.0) that enables:
- Reproducible code generation
- Autonomous vibecoding loops (error → replan → retry)
- Quality-gated outputs (sentiment, off-topic, PII detection)
- Full audit trails for compliance
- Dashboard visibility for all runs

**Success Criteria:** By end of Q2 2026, `codebase-gradle` v1.0.0 protocol is published and stable.

== Part 2: The Protocol v1.0.0

=== What is "Protocol Established"?

A **Protocol v1.0.0** means:

[cols="2,3", options="header"]
|===
| Aspect | Definition
| **Immutable Contracts** | Input/output schemas are FROZEN. No breaking changes.
| **Versioned** | Semantic versioning (x.y.z) independent of plugin version.
| **Reproducible** | Same intent + context → reproducibilityId → exact replay.
| **Auditable** | Every execution logged with timestamp, user, inputs, outputs, errors.
| **Error-Recoverable** | Automatic error classification + replanning protocol.
| **Quality-Gated** | Deterministic quality checks (sentiment, PII, off-topic).
|===

package codebase.protocol

/**
 * AssistantIntent v1.0.0
 * 
 * Immutable contract for user interaction with vibecoding assistant.
 * Version locked until v2.0.0 (backward-incompatible changes only).
 */
data class AssistantIntent(
    val protocolVersion: String = "1.0.0",
    
    // REQUIRED
    val description: String,               // "Créer 5 tests de validation"
    val domain: String,                    // "cda" | "vibecoding" | "rag"
    
    // AUGMENTATION
    val compositeContext: CompositeContext, // RAG + Graphify + code index
    
    // CLASSIFICATION
    val classification: IntentClassification, // PRO | FLASH
    
    // EXECUTION CONTROL
    val maxActions: Int = 10,              // Max actions in vibecoding loop
    val timeout: Duration = 5.minutes,     // Max execution time
    val dryRun: Boolean = false,           // Simulate without LLM calls?
    
    // REPRODUCIBILITY
    val sessionId: UUID = UUID.randomUUID(),
    val seedLlm: Long? = null,             // Fixed seed for determinism
    
    // METADATA
    val userId: String? = null,
    val projectName: String,
    val tags: List<String> = emptyList(),
) {
    init {
        require(description.isNotBlank()) { "description cannot be blank" }
        require(maxActions > 0) { "maxActions must be > 0" }
        require(timeout.toSeconds() > 0) { "timeout must be > 0" }
    }
}

enum class IntentClassification {
    PRO,   // Professional: complex, multi-step, production code
    FLASH  // Flash: simple, one-shot, quick generation
}

    val executionMetrics: ExecutionMetrics,// Success rate, errors, retries
    
    // QUALITY GATES
    val auditResults: AuditResults,        // Sentiment, PII, off-topic scores

    // CONFIDENTIALITY
    val confidentialityMask: PiiMaskingResult, // PII redactions applied
    
    // ERROR HANDLING
    val errors: List<AssistantError> = emptyList(),
    val errorRecoveries: List<ErrorRecovery> = emptyList(),
    
    // TIMESTAMPS
    val startTime: Instant,
    val endTime: Instant,
    val duration: Duration,
    
    // CONTEXT FOR NEXT RUNS
    val feedbackInputSlot: FeedbackSignal?,  // If user wants to refine
) {

}

**Implementation:**

```kotlin
data class ErrorRecovery(
    val errorCode: ErrorCode,
    val recoveryStrategy: RecoveryStrategy,
    val retryCount: Int,
    val success: Boolean,
)

enum class RecoveryStrategy {
    REPLAN,           // Change the plan, retry
    BACKOFF_RETRY,    // Exponential backoff, same plan
    FALLBACK,         // Use simpler approach
    MANUAL,           // Requires user input
}

// In VibecodingTask
fun executeWithErrorRecovery(step: ExecutionStep): Result<ExecutionStep> {
    var attempt = 0
    while (attempt < maxRetries) {
        try {
            return executeStep(step)
        } catch (e: Exception) {
            val category = classifyError(e)
            val strategy = pickRecoveryStrategy(category, attempt)
            
            attempt++
            errorRecoveries.add(ErrorRecovery(
                errorCode = mapToErrorCode(e),
                recoveryStrategy = strategy,
                retryCount = attempt,
                success = false
            ))
            
            when (strategy) {
                RecoveryStrategy.REPLAN -> {
                    val correctedPlan = replanFromStep(step.index)
                    // Update plan and continue
                }
                RecoveryStrategy.BACKOFF_RETRY -> {
                    Thread.sleep(exponentialBackoff(attempt))
                    // Continue loop
                }
                RecoveryStrategy.MANUAL -> {
                    return Failure(e)
                }
            }
        }
    }
    return Failure(MaxRetriesExceededException())
}
```

=== Reproducibility Guarantee

**Reproducibility ID = Deterministic Replay**

```kotlin
// Generate an intent
val intent = AssistantIntent(
    description = "Créer 5 tests de validation",
    domain = "vibecoding",
    seedLlm = 42,  // Fixed seed
)

// Run vibecoding
val response = vibecodeTask.execute(intent)
val reproductibilityId = response.reproductibilityId

// Later, replay exactly:
val replayedResponse = vibecodeTask.replay(reproductibilityId)

// Should be ≈ same (≤10% semantic diff acceptable)
require(
    semanticSimilarity(response.executionTrace, replayedResponse.executionTrace) >= 0.90
)
```

**Acceptable Variance:** ±10%
- LLM non-determinism (seeds help but not perfect)
- Timestamp differences (expected)
- Token usage may vary slightly
- But semantic content should be identical

**Dashboard tracks:**
- ReproductibilityId → full execution log
- Can replay at any time
- Compare N runs → reproducibility rate

=== Quality Gate Protocol

```kotlin
data class QualityGateRules(
    val protocolVersion: String = "1.0.0",
    
    // Sentiment check: Is the output motivating? (pedagogical)
    val sentimentMinScore: Double = 0.60,
    
    // Off-topic check: Is the output relevant?
    val offTopicMaxScore: Double = 0.20,
    
    // PII check: Any personally identifiable info?
    val piiMaxInstances: Int = 0,
    
    // Execution time SLA
    val executionTimeoutSeconds: Int = 300,
    
    // Effort estimation
    val maxEstimatedEffortMinutes: Int = 480,
) {
    fun validate(response: AssistantResponse): ValidationResult {
        val checks = listOf(
            CheckResult("sentiment", response.auditResults.sentimentScore >= sentimentMinScore),
            CheckResult("off_topic", response.auditResults.offTopicScore <= offTopicMaxScore),
            CheckResult("pii", response.auditResults.piiDetected.isEmpty()),
            CheckResult("execution_time", response.duration.toSeconds() <= executionTimeoutSeconds),
        )
        
        return ValidationResult(
            passed = checks.all { it.passed },
            checks = checks,
        )
    }
}
```

=== Protocol Versioning Policy

**v1.0.0 (CURRENT):**

- Output schema: FROZEN
- Error codes: FROZEN
- Quality gate rules: FROZEN
- No breaking changes until v2.0.0

- Must support v1.0.0 clients

**v2.0.0 (Future, BREAKING):**
- Only for major incompatibilities
- Only after v1.x is mature (12+ months)
- 6 month deprecation period

Before any breaking change:

1. **Issue:** Describe the breaking change + rationale
2. **RFC:** Request for Comments in GitHub Discussions

4. **Decision:** Approve or defer to v2.0.0
5. **Implementation:** Tag as v1.1.0 (additive) or plan v2.0.0

== Part 10: Quick Reference

=== Key Definitions

**Reproducibility ID:** UUID assigned to each execution, allows exact replay

**Protocol Version:** Semantic version (1.0.0) independent of plugin version

**Quality Gate:** Automated validation (sentiment, PII, off-topic, timeout)

**Error Recovery:** Automatic replan + retry when execution fails

**Audit Trail:** Complete log of who did what, when, why

**Frozen Contract:** Input/output schema locked until v2.0.0
