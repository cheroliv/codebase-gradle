== Part 1: Magic-Stick: The Foundation

=== What Magic-Stick Does

**Problem:** Linux laptops are complex. Setting up dev environment is hard. For blind/disabled users, it's nearly impossible.

**Solution:** 
- Bootable USB with Xubuntu pre-configured
- A/B partition system (atomic updates + instant rollback)
- Persistent user data across reboots
- Complete dev/diagnostic toolkit included
- Accessibility tools pre-installed (screen readers, magnifiers, etc)
- Plug any PC, boot, and work

**Target Users:**
- Blind developers
- Visually impaired individuals
- People with motor disabilities
- Neurodivergent developers
- Anyone who wants a portable Linux environment

**Status:** ✅ Already published, mature concept

---

=== Magic-Stick + Plugins Integration

**The Bridge:**

```
magic-stick (bootable)
  ↓
Xubuntu pre-configured with:
  ├─ Gradle + Kotlin
  ├─ bakery-gradle (pre-installed)
  ├─ jhipster-gradle-plugins (pre-installed)
  ├─ codebase-gradle (pre-installed)
  ├─ codex-gradle (pre-installed)
  ├─ capsule-gradle (pre-installed)
  ├─ Accessibility tools (NVDA, Magnifier, etc)
  ├─ Screen readers
  ├─ Keyboard navigation config
  └─ Voice input setup

Result:
  = Blind developer plugs in USB
  = Boots into ready-to-use dev environment
  = All tools pre-configured
  = Accessibility fully working
  = Can start coding immediately
```

== Part 2: Accessibility Architecture

=== Accessibility at Every Layer

**Layer 1: OS Level (magic-stick)**
```
- Screen reader compatible
- Keyboard-only navigation
- Configurable fonts/colors
- Audio feedback options
```

**Layer 2: Plugin Level**
```
- CLI tools (no GUI dependency)
- Gradle tasks easily scriptable
- Clear text output (parseable for screen readers)
- Error messages in plain language
```

**Layer 3: Edster SaaS (Optional)**
```
- Web dashboard (WCAG 2.1 AA compliant)
- API-first (for programmatic access)
- Dark mode / high contrast options
- Keyboard shortcuts for all features
```

**bakery-gradle:**
- [ ] CLI output readable by screen readers
