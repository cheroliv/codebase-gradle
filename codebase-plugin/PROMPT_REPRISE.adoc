== Mission

Code review globale de la boucle vibecoding pour mesurer la distance
restante avant une boucle autonome fonctionnelle de bout en bout.

Objectifs :
- Auditer TOUS les composants vibecoding existants (VibecodingGraph,
  VibecodingTask, ToolRegistry, SessionRepository, MigrationRunner)
- Identifier les gaps entre l'etat actuel et une boucle complete :
  LLM → plan → execute tools → persist state → feedback loop
- Produire un rapport AsciiDoc avec roadmap priorisee des etapes restantes
- ZERO code nouveau — lecture, analyse et rapport uniquement

ZERO agent parallele. Lecture seule sauf rapport.

== Etat d'entree

- VibecodingGraph.kt : graphe koog buildContext→classify→plan→executeTools→sendToolResult
- VibecodingTask.kt : tache Gradle vibecode (dryRun, maxActions, audit trail JSONL)
- ToolRegistry : 7 tools (read_file, write_file, edit_file, list_directory, exit, exec_shell, exec_gradle)
- VibecodingState : data class (iterations, maxActions, dryRun, planJson, error, finished)
- SessionRepository : persistance R2DBC (Jardin Secret confidentiality_level)
- TokenTracker : comptage tokens in-memory
- Dashboard : aggregation sessions
- MigrationRunner : pipeline V{n}__*.sql idempotent
- 35/35 JUnit5 PASS, 100/100 Cucumber PASS

== Questions de la review

. La boucle peut-elle tourner sans intervention humaine ?
. Quels sont les nœuds manquants dans le graphe koog ?
. Le feedback loop (error → replan → retry) est-il implemente ?
. La persistance est-elle complete (state + steps + tokens + audit trail) ?
. Le contract VibecodingState couvre-t-il tous les etats de la machine ?
. Quels sont les 3 prochains EPICs critiques pour fermer la boucle ?

== Regles absolues

- **ZERO code** : lecture et analyse uniquement
- **ZERO agent @xp ou subagent** : l'orchestrateur fait la review
- **ZERO push sans permission**
- **Rapport en AsciiDoc** : structure claire, gaps listes, roadmap priorisee
