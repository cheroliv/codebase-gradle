== Mission

Cabler le LLM (Gemini via LlmConfig) et la persistance (SessionRepository)
dans le graphe koog `VibecodingGraph` pour passer d'un executeur deterministe
de plan a un veritable agent autonome.

C'est le gap critique identifie par GLOBAL_CODE_REVIEW v13 : les composants
existent (SessionRepository 304l, TokenTracker 58l, LlmConfig 66l) mais aucun
n'est cable dans la boucle d'execution. La machine est a 70%.

Objectifs :
- Ajouter un noeud `callLLM` dans VibecodingGraph — le LLM decide quelle
  tache executer (pas de plan deterministe)
- Cabler `LlmConfig` → `GeminiConfig.resolveApiKey()` pour l'appel Gemini
- Cabler `SessionRepository.createSession()` + `updateSession()` dans le graphe
- Cabler `TokenTracker.trackPrompt()` pour mesurer la consommation reelle
- Injecter `ConnectionFactory` (R2DBC) dans VibecodingGraph constructeur
- 1 test Cucumber minimum : "LLM decides the next task autonomously"
- ZERO regression sur les 199 tests vibecoding existants
- ZERO token/secret dans le code ou les logs
