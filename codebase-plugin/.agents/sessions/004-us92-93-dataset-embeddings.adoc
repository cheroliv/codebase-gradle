=== Chunking
- Split phrase-level (par segments de lignes non-vides)
- Reassemblage en chunks de ~512 tokens max (estimation chars/3.5)
- Overlap de 50 tokens entre chunks consecutifs
- Resultat : 6 chunks pour 4 fichiers (HttpClientService:2, StringExt:1, TaskRepo:2, UserData:1)

=== Schema SQL
Tables `documents` + `chunks` avec FK `document_id` (ON DELETE CASCADE) :
- documents: id, file_name, file_path, file_size, chunk_count, created_at
- chunks: id, document_id, chunk_index, chunk_text, token_count, embedding vector(384), created_at

=== US-9.3 — Embeddings ONNX

- Tokenisation HuggingFace tokenizers + inference ONNX → float[384]
- 6 embeddings calcules et inseres via `UPDATE chunks SET embedding = ?::vector`
- Verification : 0 chunks NULL, tous dimension=384 (vector_dims)
