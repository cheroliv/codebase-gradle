== Actions

=== 1. Fix Cucumber 9_11 — OpencodeInjectorSteps "No non-empty content after [RELATIONS_GRAPHIFY]"

*Cause* : `OpencodeInjectorSteps.kt:55` — la boucle skip le contenu sur la
même ligne que l'en-tête. Quand Graphify n'a qu'une ligne de contenu, la
ligne suivante est l'en-tête Docs → break immédiat.

*Fix* : Vérifier d'abord le contenu sur la même ligne après l'en-tête
(`substringAfter(header).trim()`), puis seulement descendre dans les
lignes suivantes.

=== 2. Fix Cucumber prepare_context (3 scénarios) — strings trop spécifiques

*Cause* : Les tests vérifiaient `"[RÈGLES_EAGER] Contexte de gouvernance"`
et `"[CONTEXTE_RAG] Résultats sémantiques"` — strings qui ne sont jamais
dans le contenu réel (l'en-tête `[CONTEXTE_RAG]` est suivi des chunks RAG,
pas d'une sous-description).

*Fix* : Aligner les assertions feature avec ce que `OpencodeInjector` produit
réellement : `"[RÈGLES_EAGER]"` et `"[CONTEXTE_RAG]"` sans suffixe descriptif.

Fichiers modifiés :
- `src/test/kotlin/codebase/scenarios/OpencodeInjectorSteps.kt` (+9 lignes)
- `src/test/features/prepare_context.feature` (3 lignes modifiées)
