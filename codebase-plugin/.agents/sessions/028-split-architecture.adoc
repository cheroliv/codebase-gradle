== Objectif

Restructurer `codebase-gradle` pour adopter l'architecture standardisee
des plugins `foundry/public/` : racine consommateur minimal (dogfooding)
+ dossier `{name}-plugin/` autonome avec son propre Gradle Wrapper,
settings, version catalog, sources, tests, et gouvernance `.agents/`.

== Actions

=== Restructuration

. Suppression de `src/`, `buildSrc/`, `gradle/rag-bench.gradle.kts`,
  `gradle/anonymization.gradle.kts` de la racine.
. Suppression de `embeds.yml`, `opencode.json`, `libs/` de la racine.
. Deplacement de tous les fichiers de gouvernance (`.adoc`) dans `codebase-plugin/`.
. Creation de `codebase-plugin/` comme build Gradle independant :
   * `settings.gradle.kts` avec `foobar-resolver-convention` 1.0.0
   * `build.gradle.kts` complet : `java-gradle-plugin`, signing, publish
   * `gradle/libs.versions.toml` avec 30+ dependances (langchain4j, pgvector, cucumber...)
   * `gradlew` independant (wrapper Gradle 9.5.0)
   * `src/main/kotlin/codebase/` : CodebasePlugin.kt, rag/, benchmark/, walker/
   * `src/test/` : steps Cucumber + features + resources
   * `buildSrc/` : classes utilitaires (CodebaseYmlAnonymizer, BenchmarkConfig, etc.)
   * `.agents/` : gouvernance agent complete (INDEX, SESSIONS_HISTORY, etc.)
   * Gouvernance `.adoc` : AGENT.adoc, PROMPT_REPRISE.adoc, BACKLOG.adoc, etc.

=== Racine

. `build.gradle.kts` reduit a 6 lignes : `plugins { alias(libs.plugins.codebase) }`
   + repositories mavenLocal + mavenCentral.
. `settings.gradle.kts` : pluginManagement + rootProject.name
. `gradle/libs.versions.toml` : 2 entrees (version codebase-plugin + plugin alias)
. `gradle.properties` : conserve

   gradleJvm explicite sur 24.
