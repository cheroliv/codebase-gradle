== Résumé

Migration partielle des contrats vibecoding du package `cccp.vibecoding.contracts`
(codebase-plugin, couche N1) vers `workspace-bom` (couche N0).

== Stratégie adoptée

Seuls les fichiers **sans dépendance circulaire** vers `cccp.vibecoding.contracts.plan.Plan`
(N1) ont migrés :
- `tools/ExecShellTool.kt` → `education.cccp.contracts.vibecoding.tools`
- `tools/ExecGradleTool.kt` → `education.cccp.contracts.vibecoding.tools`
- `tools/ToolkitIsMissingException.kt` → `education.cccp.contracts.vibecoding.tools`
- `registry/ToolRegistry.kt` → `education.cccp.contracts.vibecoding.registry`

en N1 car ils dépendent de `cccp.vibecoding.contracts.plan.Plan`.

`KoogPlan.kt` adopte désormais les types `Epic`, `UserStory`, `GradleTask`
depuis `agent-contracts` (N0) au lieu de définir ses propres data classes.
